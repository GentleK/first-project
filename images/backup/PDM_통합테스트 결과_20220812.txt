		"다음 작업은 완료되었습니다. IT000000"이 아니라 승인 시 "승인되었습니다.", 반려 시 "반려되었습니다." 로 IT000000과 같이 유저가 알 필요없는 정보는 없애고 어떤 업무를 처리했는지 확실히 알 수 있도록.
		
		
	<기타>
		1. 회로자재는 승인원 늦게오는 경우가 종종 발생하여, 샘플검사서를 통하여 우선 배포할 수 있어야 한다.(현업) -> 승인원으로 진행한다.(김원태 팀장)
		2. 회로자재 등록할 때 승인원 등록->결재하면 자재가 배포되는데 결재는 누가? AS-IS는 해당 팀장이 결재했는데 To-BE는 부품관리팀장까지 결재로 바뀐다
		3. 부품의 관련 도면의 도면 목록 객체 클릭 시 반응없음 -> 개인PC 사양 문제로 기다리면 반응함
		4. 등록 팝업 창이 스스로 최소화 된다. 간헐적으로 발생 (고윤석 선임 PC)
		5. 업무흐름도의 기술문서 관리->기술문서 등록에서 결재자가 플랜트검토자(X) -> 플랜트관리자(O) 맞는지 확인하고 수정
		6. 설계변경을 하면 EBOM 붙일텐데 SINK가 안맞는 옛날도면들이 문제가 있음 -> 설계변경이 반영된 도면을 가지고 진행해야하는데 회로쪽은 바뀐게 없다.
		   UDE에서 EBOM하고 도면 SINK 안맞으면 고쳐야 하는데 이게 안되어서 옛날도면 올리면 또 문제가 발생함 -> 담당자가 퇴사하고 관리가 안되는 도면은 최신 BOM이랑 안맞을 수 밖에 없음
		   PLM에 있는 회로도와 서버에 있는 회로도 중에 어느 자료가 최신인지 모른다.  -> 자료의 신뢰성이 없다. 이러한 부분 개선할 수 있도록 고민(김원태 팀장)
		7. PCB팬텀의 ECR 프로세스에서 최초 BOM이 UDE 쪽에 없는게 맞는지? BOM Compare 기능이 있는데 마이그레이션을 하지 않았으니 비교할 자료가 없지 않나?(김원태 팀장)
		   기존 시스템은 BOM을 수정하고 도면을 수정하기 때문에 BOM이 최신이지만 UDE는 도면 부터 수정하기때문에 도면이 최신이다. 그래서 비교하게 되면 전송이 안된다(UDE)
